----

# 📅 12-Week Linux Mastery Execution Plan

Use this as a checklist in Obsidian. Do NOT skip ahead.

---

# 🗓️ Weeks 1–2: Linux Fundamentals

* [x] Understand what Linux is (Kernel vs OS)
* [x] Learn major distributions (Debian, Arch, Ubuntu, Kali)
* [x] Understand boot process (BIOS → GRUB → Kernel → systemd)
* [x] Navigate entirely using CLI
* [x] Practice basic commands daily (ls, cd, pwd, cp, mv, rm, mkdir)
* [x] Understand absolute vs relative paths

---

# 🗓️ Weeks 3–4: File System & Permissions

## File System

* [ ] Memorize directory structure (/etc, /var, /home, /usr, /proc, /dev)
* [ ] Understand file types (regular, directory, symlink)
* [ ] Understand inodes (conceptually)
* [ ] Learn mounting basics

## Permissions

* [x] Understand users & groups
* [x] Master chmod (symbolic + numeric)
* [ ] Master chown & chgrp
* [ ] Understand 755, 644, etc.
* [ ] Learn SUID, SGID, Sticky Bit
* [ ] Understand umask

---

# 🗓️ Weeks 5–6: Users & Processes

## User Management

* [x] Understand /etc/passwd
* [x] Understand /etc/shadow
* [x] Understand /etc/group
* [x] Create and manage users (useradd, adduser)
* [ ] Configure sudo

## Process Management

* [x] Understand what a process is
* [x] Learn PID & parent-child processes
* [x] Use ps effectively
* [x] Use top / htop
* [x] Use kill & killall
* [ ] Understand signals (SIGTERM, SIGKILL)
* [x] Use background jobs (&)
* [ ] Learn nice & renice

---

# 🗓️ Weeks 7–8: Networking & Logs

## Networking in Linux

* [ ] Use ip command confidently
* [ ] Understand ss & netstat
* [ ] Use ping & traceroute
* [ ] Use dig
* [x] Understand /etc/hosts
* [x] Identify listening ports
* [ ] Understand how services bind to ports

## Logs & Monitoring

* [ ] Explore /var/log
* [ ] Use journalctl
* [ ] Check authentication logs
* [ ] Monitor disk usage (df, du)
* [ ] Monitor memory (free)

---

# 🗓️ Weeks 9–10: Bash Mastery

## Bash Basics

* [ ] Write scripts with variables
* [ ] Use if/else properly
* [ ] Write for loops
* [ ] Write while loops
* [ ] Create functions
* [ ] Understand exit codes

## Bash Advanced

* [ ] Master pipes (|)
* [ ] Master redirection (> >> <)
* [ ] Understand STDOUT vs STDERR
* [ ] Use grep effectively
* [ ] Learn basic awk
* [ ] Learn basic sed
* [ ] Use xargs
* [ ] Create cron jobs

---

# 🗓️ Week 11: Services & SSH

## Services

* [x] Use systemctl to manage services
* [x] Enable/disable services at boot
* [ ] Understand what daemons are
* [ ] Inspect running services

## SSH

* [ ] Connect using ssh
* [ ] Generate SSH keys
* [ ] Configure key-based authentication
* [ ] Use scp
* [ ] Understand disabling password authentication (concept)

---

# 🗓️ Week 12: Security & Development Basics

## Security Basics

* [ ] Configure basic firewall (ufw)
* [ ] Understand iptables conceptually
* [ ] Understand Fail2ban concept
* [ ] Apply basic hardening steps
* [ ] Understand least privilege principle

## Development Basics

* [x] Compile simple C program using gcc
* [ ] Understand make (concept)
* [ ] Understand PATH variable
* [ ] Run compiled binaries

---

# 🛠️ Final Practical Projects (Do After Week 12)

* [ ] Build a Bash script to monitor open ports
* [ ] Build an automatic file backup script
* [ ] Create a log analyzer script
* [ ] Create a simple user management script
* [ ] Create a system health monitoring script

---

# 🎯 Rule

* Do not rush.
* Do not skip.
* Deep understanding > Fast progress.
* Practice daily.

If completed properly, this puts you ahead of most beginners in cybersecurity.
